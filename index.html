Let's Discuss Node.js

According to Wikipedia, Node.js is "an open source, cross-platform runtime environment for server-side and networking applications." It was initially released on May 27, 2009. It is written in the languages of C, C++, and Javascript. It can be run on OS X, Linux, Solaris, FreeBSD, OpenBSD, Microsoft Windows, and webOS. As will be discussed in further detail below, node.js, also quoted from Wikipedia, "provides an event-driven architecture and a non-blocking I/O API that optimizes an application's throughput and scalability. These technologies are commonly used for real-time web applications." To go along with that, node.js uses the Google V8 Javascript engine for coding. Groupon, LinkedIn, Microsoft, Yahoo!, Walmart, and GoDaddy are just a few examples of its use.




What is Node.js?

This brings up the question, What really is node.js? Sure, it sounds fancy and relatively new, but what does it do, and what are its advantages? Well, first off it incorporates web development in a dynamic language that runs on a VM that is fast. Second, it can handle a lot of concurrent connections. Third, it is great for creating event loops. Fourth, it is used with Javascript, which is already really popular among programmers. Lastly, it can be used on a web server, which eases communication.

What makes node different than developing with more traditional server side languages?

Now that we know what node.js is, how in the world is it different than ever other server side language? Basically, it is different because it is EVENT driven, and it uses Javascript. By event driven, I mean that is uses an asynchronous architecture, rather than the more traditional server side sychronous architecture. This means that node.js reacts to events and sends events to the database. Because of this, the language scales well. Also, because the language uses Javascript, a developer using node.js can use his Javascript knowledge both on the client and the server.

What is the concept of non blocking IO?

Non-blocking refers to code that doesn't block execution. One advantage of this is that the CPU can be kept busy and can save you memory. An example of non-blocking is how node.js uses only one thread to service all requests. Thus, the I/O is the bottleneck, not the computations. With that being said, "when requests arrive at the server, they are serviced one at a time. When the code serviced needs to query the DB for example, it sends off a request to the DB. However, instead of waiting for the response and stall, it sends the callback to a second queue and the code continues running. Now when the DB returns data, the callback gets queued in a third queue where they are pending execution. When the engine is doing nothing (stack empty), it picks up a callback from the third queue and executes it." All of this essentially means that there is no wait, or stall time. Unlike php, there is not going to be any sleep time, which translates into wasted time and space. Thus, node.js is more efficient to use in this area among others.
